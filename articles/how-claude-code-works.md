# 揭秘 Claude Code：為什麼新一代 AI 編碼代理人終於「好用了」？
> **作者：** 邱祺閔  
> **閱讀時間：** 約 4 分鐘  
> **來源：** [How Claude Code Works — PromptLayer](https://docs.promptlayer.com/why-promptlayer/how-it-works)
在 AI 工程領域，Coding Agents 正經歷一場從「玩具」到「工具」的革命。PromptLayer 的創辦人 Jared Zoneraich 在 AI Engineer 演講中深入剖析了 Claude Code 的核心架構，解釋了為什麼這類工具在近期取得了突破性進展。
> [原影片連結]
---
## 1. 核心哲學：大道至簡（Simple is Better）
Jared 指出，Claude Code 成功的關鍵不在於複雜的架構，而是回歸了**「簡單優於複雜」**的工程原則。
- **給予工具並放手（Give it tools and get out of the way）**：與其為了防止模型幻覺而建立層層疊疊的提示詞工程（Prompt Engineering），現代代理人的做法是簡化腳手架，讓模型直接與工具互動。
- **拋棄 DAG，擁抱探索**：過去開發者傾向於建立複雜的有向無環圖（DAG）來規範 AI 的行為，但現在的趨勢是依賴模型的探索能力。Jared 建議，當遇到疑慮時，應信任模型自行解決問題，而非預設所有的 if-else 邏輯。
---
## 2. 運作機制：核心的 N0 循環
Claude Code 的運作核心是一個極簡的 **While 循環**（內部可能稱為 N0）。其邏輯如下：
1. 執行 While 循環。
2. 模型根據需求呼叫工具（Tool Calls）。
3. 執行工具並將結果回傳給模型。
4. 重複此過程，直到沒有工具需要呼叫，最後才詢問使用者下一步。
這種架構利用了現代模型（如 Claude Sonnet 4.5）強大的指令遵循與自我修正能力，使其在遇到錯誤時能自動調整。
---
## 3. Bash：代理人的「萬用轉接頭」
在眾多工具中，**Bash** 被視為最重要的核心。Jared 認為「Bash 就是你所需的一切」，原因有二：
- **萬能性**：透過 Bash，代理人可以建立 Python 腳本、執行測試、配置環境，幾乎能做任何工程師在終端機能做的事。
- **訓練數據豐富**：由於全球開發者都在使用 Bash，模型對其擁有極高的熟練度，遠高於 Rust 等較少見的語言。
此外，Claude Code 捨棄了傳統的 RAG（檢索增強生成），轉而使用簡單的 **Grep** 和 **Glob** 進行程式碼搜尋，這種模擬人類行為的工具調用反而更為精確高效。
---
## 4. 最大的敵人：上下文管理（Context Management）
當上下文（Context）過長時，模型會變得「笨拙」，這是代理人開發面臨的最大挑戰。Claude Code 採用了幾種策略：
- **自動壓縮**：當上下文達到容量（如 92%）時，系統會丟棄中間部分，並摘要頭部與尾部的資訊。
- **子代理人（Sub-agents）**：針對特定任務（如讀取文件或執行測試）開啟獨立上下文的子代理人，處理完後僅回傳結果，避免主對話環境變得雜亂。
- **Handoff（移交）**：這是一種像「切換武器比裝彈快」的策略，直接開啟新執行緒並帶入必要資訊，而非在舊對話中死撐。

## 5. 與其他代理人的對比
| 代理人 | 特色 |

| **CodeEx** | 採用 Rust 開發核心，強調並行處理與事件驅動架構。 |

## 總結心得：AI 工程的新思維
### 1. 工程實踐的回歸
我們曾試圖用極其複雜的「AI 邏輯」去限制 AI，但 Claude Code 證明了好的 AI 工程其實就是好的傳統工程。簡單、扁平化、模組化的設計，配合強大的底層模型，往往比複雜的流程圖更穩定。這提醒我們，不要試圖過度設計（Over-engineer）去掩蓋模型的缺陷，因為模型進步的速度很快，今天設計的補丁可能三個月後就成了阻礙。
### 2. 上下文是稀缺資源
開發 AI 應用時，我們必須像管理內存（Memory）一樣謹慎管理上下文。使用子代理人（Sub-agents）來分擔負擔、利用 Bash 建立長期存儲的 Markdown 檔案，這些都是為了保持模型「清醒」的必要手段。
### 3. 信任模型的探索能力
Jared 提到的一個實驗令人印象深刻：他試圖幫 AI 導航網站添加標籤（Tags），結果反而讓 AI 變得困惑。這說明 AI 擁有我們意想不到的探索路徑。作為開發者，我們的角色應該是提供**精良的工具**（如 Bash 或 Diffing）和**清晰的目標**（To-do Lists），然後讓 AI 在沙盒中自由發揮。
